# 專案作業規範

1. 永遠使用正確、自然的台灣正體中文回覆。
2. 不使用中國大陸用語；不常見的專有名詞應附上英文名稱。
3. 不使用「根因」、「落地」、「魯棒性」等不自然用詞。
4. 建置、修改建置腳本或驗證編譯前，必須先閱讀並遵循 `BUILDING.md`。
5. 標準建置驗證指令為 `cmd /c build.cmd x64`，不得自行改用其他建置方式取代。
6. 建置前必須確認 `wcap-tw-<架構>.exe` 與舊版 `wcap-<架構>.exe` 均未在執行；不得擅自終止使用者正在執行的程式。
7. 所有編輯紀錄都必須完整寫入 Git 提交訊息，包含修改原因、修改內容與驗證結果。
